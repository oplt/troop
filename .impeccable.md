## Design Context

### Users
Development teams at companies — engineering leads, ICs, ops-minded managers. Mixed session shapes: long focused orchestration sessions (building agent hierarchies, reviewing runs, approving outputs) AND quick glance check-ins between IDE work. Product lives in browser tabs alongside GitHub, Linear, and the IDE.

### Brand Personality
**Confident · editorial · utility-first.**

User said "modern / clean / professional." Those words are dead on arrival — every SaaS tool says them. Translating them into design-actionable adjectives:

- **Confident** — opinionated defaults, no timid styling, commit to choices. Orange accent is rare and earned, not sprinkled.
- **Editorial** — information density done with typographic hierarchy, not boxes and borders. Feels like a well-edited product page, not a dashboard template.
- **Utility-first** — nothing decorative for its own sake. Every pixel serves observing, controlling, or approving agent runs.

Emotional goals: user should feel **in command** of their agent fleet, and **trust** what they're seeing. Streaming activity should feel alive, not noisy.

### Aesthetic Direction

**Inheriting and sharpening the existing Mistral-inspired warm-brutalist direction.** Keep:

- `borderRadius: 0` everywhere. No rounded rectangles.
- Mistral Orange (`#fa520f`) as sole brand accent — rarity = power.
- Warm ivory (`#fffaeb`) and cream (`#fff0c2`) surfaces in light mode. Warm-toned dark mode (`#1f1f1f` / `#2a2020`), not cold blue-black.
- Light + dark toggle retained. Light is default daytime. Dark is terminal/ops mode.

Tune:

- Arial is a placeholder — swap to a non-reflex grotesque.
- Fewer, heavier divisions. Rules, not boxes. Borders, not cards-in-cards.
- Hierarchy carried by type scale contrast (≥1.25 ratio) and spacing, not color.

**Typography stack** (all non-reflex, all have strong identity):

- **Neue Montreal** (Pangram Pangram) — body + UI. Grotesque with slight warmth, not Inter/DM Sans territory. Free for personal, paid for commercial.
- **Editorial New** (Pangram Pangram) — display/serif for hero headings on marketing, run summaries, agent bios. Adds editorial voice where it counts.
- **Monaspace Neon** (GitHub, free, OFL) — mono for run streams, code, agent IDs, timestamps. Designed for code, has texture healing. Makes streaming logs feel like a Bloomberg/ops terminal, not a generic code block.

If Pangram licensing is blocker: fall back to **Switzer** (Fontshare, free OFL) for body, **Gambarino** (Fontshare, free) for display serif. Same intent.

### The Differentiator

**The way runs stream.** This is the one thing users should remember. Every design decision must serve it:

- Streaming run view is the hero surface. Monospace-dominant. Temporal flow visible: token-by-token or event-by-event reveals.
- Motion is functional — new events enter with a 150ms fade + 4px translate, never bounce. Exponential ease-out.
- Status changes (queued → running → complete → failed) communicated through weight and color, not icons.
- Approvals and intervention points are highlighted with orange accent, the only place orange appears on that screen.
- Run Inspector, Activity Audit, Execution Insights all feed back to the stream metaphor — time flows top-down, present is always visible.

### Design Principles

1. **Editorial over dashboard.** Layouts use columns, rules, and type hierarchy like a magazine. Not rows-of-identical-cards SaaS grid.
2. **One accent, earned rarely.** Orange appears for actions requiring human attention (approvals, escalations, errors). Everywhere else: neutral warm tones + type weight.
3. **Streaming is alive.** Live data surfaces have real motion and temporal shape. Static data has none — this contrast teaches the interface.
4. **Dense but calm.** Ops users want information density. Deliver it through tight type scale and rhythm, not by cramming boxes. Line length capped at 75ch for prose; tabular data ignores this.
5. **Commit to borderRadius: 0.** No softening. The brutalist edge is the brand signal. Pair with warm surfaces so it reads intentional, not cold.

### Accessibility

No explicit user requirements given. Default baseline:

- WCAG 2.2 AA contrast on all text (4.5:1 for body, 3:1 for large text and UI elements).
- Respect `prefers-reduced-motion` — streaming reveals become instant snaps, no animation.
- Focus rings visible in both themes, use accent orange with sufficient contrast outline.
- Color never sole signal — status uses weight + position + color, not color alone.
